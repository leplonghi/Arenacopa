Criar Conta no Google Play Console

⏱ 20–30 min · US$ 25 (taxa única)
🌐

Acessar o Play Console

Abra play.google.com/console e clique em "Começar agora". Faça login com sua conta Google.

👤

Tipo de conta: Pessoa Física

Para o Arena Cup, escolha Pessoa Física / Individual. Preencha nome, endereço e telefone.

ℹ️

Dica: Use "Empresa" apenas se tiver CNPJ. Pessoa Física é suficiente para apps independentes.

💳

Pagar a taxa de registro

Clique em "Pagar taxa de registro" → US$ 25 via cartão. Essa taxa é paga apenas uma vez e libera a publicação de quantos apps você quiser para sempre.

✅

Aceitar os termos e aguardar aprovação

Após o pagamento, aceite os termos de serviço. A aprovação leva entre 1 a 3 dias úteis. Você receberá um e-mail de confirmação do Google.

⚠️

Atenção: Não tente publicar antes da conta ser aprovada.

📷 Play Console — Tela de boas-vindas após aprovação
▶ Google Play Console play.google.com/console
✓
Conta aprovada — Bem-vindo ao Play Console
Você pode criar e publicar apps Android

Gerar Keystore (Chave de Assinatura)

⏱ 5 min · Faça UMA VEZ e guarde para sempre
🔑

MUITO IMPORTANTE: O keystore é a "identidade" do seu app. Se você perdê-lo, nunca mais poderá atualizar o app na mesma conta. Faça backup em pelo menos 2 locais (ex: pen drive + Google Drive). Nunca commite no Git nem compartilhe com ninguém.

⌥

Opção A: Gerar via linha de comando (keytool)

Abra o CMD ou PowerShell e execute:

keytool -genkey -v ^ -keystore arena-cup-release.keystore ^ -alias arena-cup-key ^ -keyalg RSA ^ -keysize 2048 ^ -validity 10000 # Quando solicitado, informe: # Senha: ArEnAcUp@2026! (guarde bem) # Nome: Eduardo Longhi # Org: Antigravity # Cidade: Sao Luis # Estado: MA # Pais: BR
🔨

Opção B: Gerar via Android Studio (mais visual)

Menu: Build → Generate Signed Bundle / APK... → Create new keystore

Configurar no android/app/build.gradle

android { signingConfigs { release { storeFile file("C:/Users/eduar/arena-cup-release.keystore") storePassword "SUA_SENHA" keyAlias "arena-cup-key" keyPassword "SUA_SENHA" } } buildTypes { release { signingConfig signingConfigs.release minifyEnabled false } } }
⚠️

Segurança: Adicione *.keystore ao seu .gitignore. Nunca suba a senha para o GitHub!

4

Gerar o Android App Bundle (AAB)

⏱ 10–20 min · Formato exigido pelo Play Store
📦

AAB vs APK: O Play Store exige .aab desde 2021. É menor que o APK e o Google otimiza automaticamente para cada dispositivo. Use APK apenas para testes diretos no celular.

🏗️

Passo 1: Build do frontend (Vite)

cd "C:\Users\eduar\OneDrive\Desktop\Antigravity\ArenaCopa\Arenacopa" # Gerar build de producao npm run build # Resultado esperado: # dist/index.html # dist/assets/ (JS, CSS, imagens)
📲

Passo 2: Sincronizar com o Android (Capacitor)

npx cap sync android # Esperado: # Copying web assets from dist to android/app/src/main/assets/public # Updating AndroidManifest.xml # update android - Finished
🔨

Passo 3: Gerar o AAB assinado no Android Studio

Build → Generate Signed Bundle / APK...

Menu superior do Android Studio

Selecionar "Android App Bundle"

Primeira opcao. Clique em Next.

Selecionar o keystore

Browse ate arena-cup-release.keystore → alias: arena-cup-key → inserir senhas

Build Variant: "release" → Finish

AAB gerado em: android/app/release/app-release.aab

Alternativa via terminal (cmd no Windows):

cd "C:\...\Arenacopa\android" gradlew.bat bundleRelease # Arquivo em: # app\build\outputs\bundle\release\app-release.aab

Política de Privacidade

⏱ 15 min · Obrigatório (app coleta dados)
⚠️

Obrigatório: O Arena Cup usa login Google (coleta email, nome, foto). Uma URL de Política de Privacidade é obrigatória.

📄

Hospedar no Firebase Hosting

Crie public/privacidade.html no projeto e faça deploy. URL será:

🔗 https://arenacopa-web-2026.web.app/privacidade
🔗

Ou usar um gerador gratuito

Sites como privacypolicygenerator.info ou app-privacy-policy-generator.nisrulz.com geram uma política profissional em minutos. Hospede o HTML gerado no Firebase Hosting.

Pontos obrigatórios na política do Arena Cup:

  • ✓
    Quais dados são coletados (email, nome, foto via Google Auth)
  • ✓
    Como são usados (autenticação e exibição no app)
  • ✓
    Que usa Firebase (Google) para armazenar dados
  • ✓
    Como o usuário pode excluir a conta (ex: contato por email)
  • ✓
    Que o app não vende dados a terceiros

Fazer Upload do AAB e Teste Interno

⏱ 20–30 min
📤

Criar versão de Teste Interno

Play Console → Testes → Teste interno → Criar nova versão

Clique em "Fazer upload" e selecione o app-release.aab gerado no Passo 4.

👥

Adicionar testadores

Adicione emails de Google Account de pessoas que vão testar. Eles recebem um link para instalar o app pelo Play Store em modo de teste, sem precisar de revisão do Google.

ℹ️

Testes internos são liberados em minutos. Valide que o app abre, navega bem, o login funciona e as notícias aparecem.

🐛

Verificar avisos obrigatórios

O Play Console mostra itens 🔴 (obrigatórios) e 🟡 (recomendados). Resolva todos os vermelhos antes de avançar para produção.

⚠️

App ID: O app está com o ID genérico do Lovable. Mude para com.antigravity.arenacup agora — depois da primeira publicação não é mais possível alterar.

Publicar na Produção

⏱ 10 min para enviar · 1–7 dias para revisão do Google

Lançamento → Produção → Criar nova versão

No menu lateral esquerdo do Play Console

Upload do AAB (mesmo do teste ou um novo)

Se fez correções, gere um novo AAB com versionCode maior (ex: 2). O versionCode deve sempre aumentar.

Preencher "Notas da versão"

"Versão inicial — Acompanhe jogos, crie bolões e leia notícias da Copa 2026 em tempo real!"

Definir países de disponibilidade

Recomendamos "Todos os países" ou pelo menos: Brasil, Portugal, Estados Unidos, Argentina, Espanha.

Clicar em "Enviar para revisão"

O Google revisará em 1 a 7 dias. Você receberá e-mail de aprovação ou de problemas.

🚀

Estratégia de lançamento: Publique com pelo menos 2 semanas de antecedência antes do início da Copa (11 de junho de 2026). Isso dá tempo para resolver possíveis rejeições e acumular os primeiros usuários orgânicos.
